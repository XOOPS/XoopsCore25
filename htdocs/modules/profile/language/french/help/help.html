Aide : Profile

DESCRIPTION

Le module Profil permet de gérer les champs de profil personnalisés des utilisateurs.

INSTALLER / DÉSINSTALLER

Aucune mesure spéciale n'est nécessaire, suivez le processus d'installation standard - extraire le dossier du module /profile dans le répertoire « modules ». Installez le module via Admin -> Module Système -> Modules.

Un tutoriel détaillé d'installation des modules dans Xoops est disponible à partir de ce lien Manuel d'Utilisation de XOOPS

MODE D'EMPLOI

Pour utiliser ce module, vous devez :

  • Configurer vos préférences pour le module (voir « Préférences ») et optionnellement l'affichage d'un bloc si vous désirez l'utiliser (voir « Blocs »).
  • Éditer les catégories existantes ou en ajouter de nouvelles,
  • Éditer des champs de saisie existants ou en ajouter des nouveaux. Ici vous pouvez préciser quels champs seront visibles dans quelles catégories, et s'ils doivent être visibles lors du processus d'inscription d'un utilisateur,
  • Définir l'ordre des différentes étapes du processus d'inscription,
  • Et enfin, vous pouvez définir des autorisations pour les champs individuels - ceux qui sont modifiables, ceux qui sont consultables.

Mesures anti-spam

Pour réduire les inscriptions de spam, procédez comme suit :

  • allez dans l'Admin du module Protector, allez dans Préférences, puis cochez cette option : « Stop Forum Spam » Cela vérifiera les comptes des spammeurs enregistrés dans la base de données de www.stopforumspam.com.
    Choisir « Bannir l'IP (sans limite) »

  • dans /class/captcha/config.php, assurez-vous que le mode défini est bien « texte » :

    return $config = array(
    'disabled' => false, // Désactive le CAPTCHA
    'mode' => 'text', // mode par défaut, vous pouvez choisir 'text', 'image', 'recaptcha' (nécessite api key)
    'name' => 'xoopscaptcha', // Nom du CAPTCHA
    'skipmember' => true, // Passer le contrôle CAPTCHA pour les membres
    'maxattempts' => 10, // Maximum de tentatives pour chaque session
    );

  • Dans le module Profile, allez dans Admin et à l'étape de base situé à :
    /modules/profile/admin/step.php?id=1
    définir « Enregistrer après l'arrêt » à « Non »


  • Dans les Préférences de Profile, réglez l'option « Utiliser le Captcha après la deuxième étape d'enregistrement » sur « Oui » (C'est la valeur par défaut)
